# CombatCoach 安装与使用手册

> 本插件是 WoW 12.0.x 的**实时技能计量器**：打开窗口即持续采集并实时显示你打出的技能占比、DPS/HPS；战斗结束后可按需点「分析」，对某个分段做输出/治疗/覆盖率的复盘。
> 它遵循"专一"原则——**采集与分析分离**：采集层如实呈现你打了什么（永不过时、不评判），分析层只在有客观依据时才出声。**不做**团队排名，也不替代 Details!/WarcraftLogs。
> 支持坦克（T）/输出（DPS）/治疗三种角色。

---

## 1. 它能做什么 / 不能做什么

数据全部来自游戏的**战斗日志**（Combat Log），这决定了几条硬边界：

**能做**
- 窗口常驻，**实时**显示你（含宠物/图腾/守护等召唤物）打出的技能占比、DPS/HPS。
- 分段查看：当前战斗 / 打开至今总计 / 历史各场，可切换。
- 战斗后按需分析：过量治疗%、你在主动维持的光环覆盖率、死亡等**有客观依据**的复盘。

**不能做**（WoW 插件的硬限制，非缺陷）
- 拿不到别人精确的能量/法力和技能冷却剩余。
- 只对"日志范围内"的单位负责——出视野/切目标会丢数据。
- **不评判"你的技能占比合不合理"**：这需要按专精×英雄天赋细分的外部基准（SimC/WCL），我们没有可靠来源，硬做只会误导，故不做。

## 2. 安装

插件本体位于本目录的 `CombatCoach/` 文件夹（与 `CombatCoach.toc` 同级）。

### 2.1 找到魔兽的插件目录

把 `CombatCoach` 整个文件夹复制到对应版本的 `Interface\AddOns\` 下：

| 版本 | 目标路径 |
|---|---|
| 正式服（Retail） | `World of Warcraft\_retail_\Interface\AddOns\` |
| 测试服（PTR） | `World of Warcraft\_ptr_\Interface\AddOns\` |

- **Windows** 示例：`C:\Program Files (x86)\World of Warcraft\_retail_\Interface\AddOns\CombatCoach\`
- **macOS** 示例：`/Applications/World of Warcraft/_retail_/Interface/AddOns/CombatCoach/`

### 2.2 目录结构应如下

```text
Interface/AddOns/CombatCoach/
  CombatCoach.toc     <- 插件清单
  Init.lua
  Core.lua
  Constants.lua
  Utils.lua
  Config.lua
  Segment.lua
  CombatLog.lua
  Metrics.lua
  Analyzer.lua
  Commands.lua
  Data/Defaults.lua
  UI/MeterFrame.lua
  UI/ReportFrame.lua
```

> 关键：`CombatCoach.toc` 必须直接位于 `AddOns\CombatCoach\` 下，不要多套一层文件夹。

### 2.3 在游戏中启用

1. 启动游戏，进入**角色选择界面**。
2. 左下角点击 **「插件」（AddOns）**，勾选 **CombatCoach**。
3. 若提示版本不符，勾选 **「加载过期插件」**。
4. 进入游戏，加载成功后聊天框打印：
   `CombatCoach: v0.2.0 已加载，输入 /cc 打开实时计量窗口。`

---

## 3. 快速上手（30 秒）

1. 输入 `/cc` 打开计量窗口——**窗口一开就开始采集**。
2. 打怪，窗口里的技能条会**实时刷新**，第一下就能看到数字跳动。
3. 顶部按钮：`伤害`/`治疗` 切换口径；`分段▾` 在 当前战斗→总计→历史 之间切换；`分析` 对当前查看的分段做复盘。
4. 想停止采集：再按 `/cc` 或点窗口关闭按钮——**关窗即停采，数据保留**。

> 打了几秒窗口还是 0？那是采集异常（不是你没输出），窗口会显式提示，用 `/cc debug` 定位。

---

## 4. 命令一览（`/cc`）

| 命令 | 作用 |
|---|---|
| `/cc` | 打开/关闭实时计量窗口（窗口开=采集，关=停止） |
| `/cc analyze` | 分析当前查看的分段，弹出分析视图 |
| `/cc reset` | 清空本次采集的所有分段数据 |
| `/cc debug` | 打印采集诊断（定位为何没有数据） |

> `/combatcoach` 是 `/cc` 的完整别名。

---

## 5. 窗口怎么看

**计量窗口（主体）**
- 顶部：当前口径的 DPS 或 HPS + 段时长。
- 中部：技能条列表，按伤害/治疗降序，带占比与背景条，实时刷新。
- 段名旁若出现红色 ⚠，表示这段没采到你的数据，可能异常。

**分析视图（点「分析」弹出）**
- 顶部主指标（按角色显示 DPS/HPS/DTPS + 过量治疗）。
- 改进点：按严重度红/黄/灰排序。
- 本段占比：治疗看治疗构成，其余看输出构成——数值来自实战日志，每段不同。

---

## 6. 分析层做了什么（以及为什么克制）

初版**只保留有客观依据、不依赖预写技能表、不靠拍脑袋阈值**的判断：

- **死亡**（红）：死了就是问题，无需任何基准。
- **过量治疗%**（治疗）：日志原生字段，"溢出高=效率低"是通识；阈值仅作参考，可在 `Data/Defaults.lua` 调。
- **维持光环覆盖率偏低**：不预设"哪个技能该维持"，而是从本段推断——你反复施放/刷新（说明在主动维持）的光环，若覆盖率明显偏低，就是"想维持却断了"的真实信号。对任何职业、任何赛季都成立，不会过时。

**明确不做**：CD 该用几次、技能占比是否合理、坦克减伤够不够——这些需要外部基准，我们没有可靠来源，故不冒充"知道你该怎么打"。占比只如实呈现，判断留给你自己（与 Details! 同理）。

## 7. 常见问题（FAQ）

**Q：打了几秒 DPS 还是 0？**
- 这是采集异常，不是分析结果。窗口会显式提示，输入 `/cc debug` 看 `selfGUID`/CLEU 计数定位。

**Q：为什么不告诉我技能占比合不合理？**
- 因为"合理值"随 专精×英雄天赋×场景 变化很大，只存在于 SimC/WCL 等外部数据，插件内没有可靠来源。硬给会误导，所以我们只如实呈现占比。

**Q：召唤物/图腾的伤害算吗？**
- 算。通过 `SPELL_SUMMON` 自动把你的召唤物纳入归属，其伤害计入你的总量。

**Q：数字和 Details! 对不上？**
- 本插件只统计"日志范围内的你自己"，吸收/多段用近似算法，定位是趋势与手法复盘，不是精确到个位的伤害表。

**Q：关掉窗口数据会没吗？**
- 关窗只停止采集，已采数据保留，重开可继续查看/切分段。`/cc reset` 才会清空。

**Q：会影响性能吗？**
- 战斗日志只在窗口打开时注册，热路径只做数值累加，聚合/分析只在你点「分析」时跑一次。

---

## 8. 卸载

- 临时停用：角色选择界面的插件列表取消勾选 CombatCoach。
- 彻底删除：删除 `Interface\AddOns\CombatCoach\` 文件夹。存档 `WTF\...\SavedVariables\CombatCoachDB.lua` 可一并删除。

## 9. 打包与发布（开发者）

```bash
./package.sh --check    # 只校验（.toc 引用完整性、版本一致）
./package.sh            # 打包到 plugin/dist/CombatCoach-<版本>.zip
```
